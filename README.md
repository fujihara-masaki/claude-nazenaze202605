# なぜなぜ分析ツール

品質管理・障害分析・業務ミス分析のための事実整理型なぜなぜ分析 Web GUI ツールです。  
**個人を責めるのではなく、手順・ルール・組織構造などの仕組み側の問題**を整理することを設計思想としています。

---

## 起動方法

```bash
npm install
npm run dev
```

ブラウザで `http://localhost:5173` を開いてください。

---

## 主な機能

| 機能 | 説明 |
|------|------|
| 分析テーマ・事象概要入力 | 何について分析するかを記述 |
| 5W1H整理 | When / Where / Who / What / How / Impact を整理 |
| なぜなぜ分析（最大5階層） | Why を繰り返し根本原因に迫る |
| 観点別原因候補メモ | 手順・役割・教育・業務負荷・ルール・システム・組織の7観点 |
| 事実/推測/要確認ラベル | 各記述の確実性を色分け管理 |
| 暫定/恒久/再発防止策入力 | 対策を3段階で整理 |
| 個人責任チェックリスト | 分析が個人批判に偏っていないかを確認 |
| Markdownコピー | 分析結果を業務報告書向けにMarkdown形式でコピー |
| サンプルデータ | 操作イメージを掴むためのサンプルを一発読み込み |
| localStorage自動保存 | ブラウザを閉じても入力内容が保持される |

## ラベルの使い方

| ラベル | 用途 |
|--------|------|
| **事実** | 記録・ログ・目撃者証言など証拠のある情報 |
| **推測** | 状況から推測される情報（確認が望ましい） |
| **要確認** | 不明確で追加調査が必要な情報 |

---

## なぜなぜ分析の進め方

1. **テーマ・事象概要** を記入する
2. **5W1H** で事実を整理する
3. **Why 1** に「なぜそうなったか」を記入する
4. Why 1 に対して **7つの観点**（手順/役割/教育/負荷/ルール/システム/組織）から原因候補をメモする
5. Why 1 の原因を「なぜ？」と掘り下げて **Why 2〜5** を追加する
6. 各Whyに観点別メモを繰り返す
7. **対策**（暫定・恒久・再発防止）を記入する
8. **個人責任チェックリスト**でレビューする
9. **Markdownコピー**で業務報告書に転記する

## ファイル構成

```
src/
├── components/
│   ├── AnalysisForm.tsx        # 左パネル：フォーム全体
│   ├── ThemeSection.tsx        # テーマ・事象概要
│   ├── FiveW1HSection.tsx      # 5W1H入力
│   ├── WhyTree.tsx             # Whyツリー管理
│   ├── WhyNode.tsx             # 単一Whyノード（観点別メモ含む）
│   ├── CountermeasureSection.tsx # 対策入力
│   ├── CheckList.tsx           # 個人責任チェックリスト
│   ├── PreviewPanel.tsx        # 右パネル：プレビュー・コピー
│   └── LabelBadge.tsx          # 事実/推測/要確認トグル
├── hooks/
│   └── useLocalStorage.ts      # localStorage 自動保存（debounce付き）
├── types/
│   └── analysis.ts             # 型定義
├── utils/
│   ├── markdown.ts             # Markdown生成ロジック
│   └── sampleData.ts           # サンプルデータ
├── App.tsx
├── main.tsx
└── index.css
```
